Switches

Switches are successors to a network device called a hub, so in simple terms a switch is a better version of a hub. One may ask what makes a switch different from a hub? Here is a list of things the switch does that differentiates it from a hub:

  • Switches work on the second layer of the OSI model whilst hubs work on the first
  • Switches have collision domains on each port whilst hubs had one collision domain
  • Switches have more ports compared to hubs
  • Switches filter packets whilst hubs don't
  • Switches use full duplex transmission mode whilst hubs use half duplex

What do switches do?
A switch receives frames and directs them to their destination.
A switch learns a mac address when a device sends it a frame then it stores that mac address in something known as the CAM table.
If the mac address is not found on the CAM table the switch floods the first frame to all ports.
Switches are a plug and play.

WAP

WAP stands for Wireless Wireless Point.
Just like the name suggests this is a device that make it possible for devices in a network to communicate wirelessly It is a layer 2 device so it uses frames, it sends frames wirelessly over 2.4GH and 5GH radio frequences.
WAP folows wifi standard 802.11

Router

A router is simply a network device that connects networks, it's a GATEWAY between networks.
Unlike the switches and WAP a router is a layer 3 device, which means it doesn't use frames but something called packets.
Instead of mac addresses routers use IP addresses, which are found in layer 3.
Switches are not plug and play they need configurations to work.
